# Lambda Return and the T+1 Requirement

This document explains how lambda returns work in Dreamer‑V3 and why the algorithm requires a value sequence of
length **T+1** even though the imagined trajectory contains only **T** rewards. The goal is to preserve long‑horizon
credit assignment while keeping the critic stable and the actor’s advantages well‑behaved.

---

## What a Lambda Return Represents

A lambda return blends two sources of information:

- **One‑step TD targets**, which rely heavily on the critic’s prediction of the next state’s value.
- **Monte‑Carlo rollouts**, which rely entirely on the sequence of rewards.

Lambda returns interpolate between these extremes using a parameter λ. Lower λ values trust the critic more, while
higher λ values trust the imagined rewards more. This blend helps stabilize training by reducing variance without introducing too much bias.

---

## Why Lambda Returns Need T+1 Values

A lambda return for step t always depends on the value of the **next** state. With T imagined rewards, the recurrence requires:

- V(s₁), V(s₂), …, V(s_T) from the critic applied to imagined states.
- One additional value to bootstrap the final step.

This produces a value sequence of length **T+1**. The first entry is unused but required for shape consistency, and the final entry provides the bootstrap value needed to compute the last return. Without this extra value, the recurrence
cannot propagate backward correctly.

---

## How Dreamer‑V3 Produces the Required Values

Dreamer‑V3 imagines trajectories entirely in latent space. Each imagined step produces:

- A reward decoded from the latent state.
- A new latent state generated by the world model.
- A critic value for that latent state.

The critic is evaluated on all T imagined states to produce V(s₁) through V(s_T). A final bootstrap value is computed
from the last latent state, giving the required T+1 values. These values are then passed to the lambda return function
along with the T rewards.

---

## Why Real‑Environment Truncation Does Not Affect This

Truncation only affects real‑environment rollouts. When the environment ends early, the world model resets its latent
state and begins a new imagined trajectory from the next real observation. Imagination itself always produces a full
horizon of length T, so the lambda return always receives a complete value sequence of length T+1. This separation
ensures that the algorithm remains stable even when episodes end unpredictably.

---

## Summary

- Lambda returns blend TD and Monte‑Carlo targets to stabilize long‑horizon learning.
- The recurrence requires the value of the next state at every step, which means T imagined rewards require T+1 values.
- Dreamer‑V3 naturally produces these values through critic evaluations and a final bootstrap.
- Real‑environment truncation does not interfere with this process because imagination is independent of episode length.

This structure keeps the critic consistent, the actor’s advantages meaningful, and the overall training process stable.
